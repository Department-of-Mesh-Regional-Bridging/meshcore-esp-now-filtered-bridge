Crossband Filtered Bridge Project

An Organization Dedicated to Healthy, Sovereign, and Scalable MeshCore Networks

Mission

To enable intentional, low‑noise communication between independent MeshCore regions operating on different frequency settings—while filtering out common non‑essential flood traffic and preserving the long‑term health, sovereignty, and scalability of the mesh ecosystem.

Overview

The Crossband Filtered Bridge is a dual‑radio ESP‑NOW system that links two MeshCore regions without forcing them to share the same RF environment.It selectively forwards only meaningful traffic, preventing cross‑region noise, advert storms, and public‑channel chatter from overwhelming neighboring meshes.

This project promotes regional autonomy, mesh hygiene, and cooperative inter‑mesh communication.

Key Features

Customizable Packet Filtering

CLI‑controlled filtering of:

Adverts

Main Public channel

Up to 16 hashtag channels

Supports fine‑grained control of what crosses the bridge(e.g., #sacsfbridge for inter‑mesh threads)

Cross‑Frequency Bridging

Enables communication between meshes operating on different frequency settings

Each mesh preserves:

Optimal topography‑based frequency choices

Sparse vs. dense environment tuning

Link budget advantages unique to their region

Mesh Sovereignty & Health

Prevents unnecessary cross‑region traffic

Reduces advert floods and public‑channel noise

Encourages smaller, healthier meshes connected by intentional message threads

Helps avoid mesh threshold collapse in large, overloaded networks

Private messages

Why This Matters

Modern MeshCore deployments face challenges as they grow:

Advert storms

Public‑channel saturation

Region bleed‑through

Frequency mismatches

Mesh collapse due to excessive node count

The Crossband Filtered Bridge solves these by acting as a selective gateway, not a repeater.It forwards only what should cross, preserving the integrity and performance of each region.
